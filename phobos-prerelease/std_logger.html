std.logger

Implements logging facilities.

Message logging is a common approach to expose runtime information of a program. Logging should be easy, but also flexible and powerful, therefore D provides a standard interface for logging.

The easiest way to create a log message is to write import std.logger; Log("I am here"); this will print a message to the stdio device. The message will contain the filename, the linenumber, the name of the surrounding function and the message. Five additional functions exists, that will log messages to the with the corresponding LogLevel. These functions are called Info, Warning, Error, Critical and Fatal. All log functions work with all Logger classes.

Example:
Log("Logging to the defaultLogger with its default LogLevel");
Info("Logging to the defaultLogger with its Info LogLevel");
Warning("Logging to the defaultLogger with its Warning LogLevel");
Error("Logging to the defaultLogger with its Error LogLevel");
Critical("Logging to the defaultLogger with its Critical LogLevel");
Fatal("Logging to the defaultLogger with its Fatal LogLevel");

auto fileLogger = new FileLogger("NameOfTheLogFile");
fileLogger.Log("Logging to the fileLogger with its default LogLevel");
fileLogger.Info("Logging to the fileLogger with its Info LogLevel");
fileLogger.Warning("Logging to the fileLogger with its Warning LogLevel");
fileLogger.Error("Logging to the fileLogger with its Error LogLevel");
fileLogger.Critical("Logging to the fileLogger with its Critical LogLevel");
fileLogger.Fatal("Logging to the fileLogger with its Fatal LogLevel");

Messages are logged if the LogLevel of the log message is greater equal than the LogLevel of the used Logger and the global LogLevel. To assign the LogLevel of a Logger use the logLevel property of the logger. The global LogLevel is managed by the LogManager. It can be changed by assigning the globalLogLevel property of the LogManager.

To customize the logger behaviour, create a new class that inherits from the Logger class and implements the writeLogMsg method.

Example:
class MyCustomLogger : Logger {
	override void writeLogMsg(const string fn, const int l,
			const string fun, const string prettyFun,
			const LogLevel lvl, const string msg)
	{
		// log message in my custom way
	}
}

auto logger = new MyCustomLogger();
logger.Info("Awesome log message");

In order to disable logging, pass DisableLogger as a version argument to the D compiler.

enum LogLevel: int;
There are five usable logging level. These level are Info, Warning, Error, Critical and Fatal. Ever log message and every Logger has a LogLevel associated with it.

Default
If no LogLevel is passed to the log function this level indicates that the current level of the Logger is to be used.

Info
This level is used to display information about the program.

Warning
Warnings about the program should be displayed with this level.

Error
Information about errors should be logged with this level.

Critical
Messages that inform about critical errors should be logged with this level.

Fatal
Log messages that describe fatel errors should use this level.

abstract class Logger;
This class is the base of every logger. In order to create a new kind of logger a derivating class needs to implementation the method writeLogMsg.

this(LogLevel lv = LogLevel.Info);
The constructor of the abstract logger has one arguments. The argument defines the LogLevel of the Logger. By default the LogLevel is set to Info. If a custom constructor is needed the selected LogLevel needs to be passed to the super constructor.

void logMessage(const string fn, const int l, const string fun, const string prettyFun, const LogLevel lvl, const string msg);
This method is the entry point into each logger. It compares the given LogLevel with the LogLevel of the Logger and the global . If the passed LogLevel is greater or equal to both the message and all other parameter are passed to the abstract method writeLogMsg.

final const pure nothrow @property LogLevel logLevel();
Get the LogLevel of the logger.

final pure nothrow @property void logLevel(const LogLevel lv);
Set the LogLevel of the logger.

class StdIOLogger: std.logger.Logger;
This Logger implementation writes log messages to the systems standard output. The format of the output is: FileNameWithoutPath:FunctionNameWithoutModulePath:LineNumber Message.

class FileLogger: std.logger.Logger;
This Logger implementation writes log messages to the associated file. The name of the file has to be passed on construction time. If the file is already present new log messages will be append at its end.

final void writeLogMsg(const string fn, const int l, const string fun, const string prettyFun, const LogLevel lvl, const string msg);
The messages written to file have the format of: FileNameWithoutPath:FunctionNameWithoutModulePath:LineNumber Message.

@property File file();
The file written to is accessible by this method.

class LogManager;
The static LogManager handles the creation and the release of instances of the Logger class. It also handels the defaultLogger which is used if no logger is manually selected. Additionally the LogManager also allows to retrieve Logger by there name. An StdIOLogger is assigned to be the default Logger.

static @property Logger defaultLogger();
This method returns the default Logger.

static @property void defaultLogger(Logger log);
This method sets the default Logger. Example 
LogManager.defaultLogger = new StdIOLogger();
Log("A message from the defaultLogger");

static @property LogLevel globalLogLevel();
This method returns the global LogLevel.

static @property void globalLogLevel(LogLevel ll);
This method sets the global LogLevel.
