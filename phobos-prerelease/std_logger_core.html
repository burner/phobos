std.logger.core

Implements logging facilities.

Message logging is a common approach to expose runtime information of a program. Logging should be easy, but also flexible and powerful, therefore D provides a standard interface for logging.

The easiest way to create a log message is to write import std.logger; log("I am here"); this will print a message to the stdio device. The message will contain the filename, the linenumber, the name of the surrounding function, and the message.

By default only one Logger exists, this is the defaultLogger. In order to use this Logger simply call the free standing log functions like:
  • log
  • trace
  • info
  • warning
  • critical
  • fatal
    • The defaultLogger will log to stdout and has a default LogLevel of LogLevel.all.

    Additional Logger can be created by creating a new instance of the required Logger. These Logger have the same methodes as the defaultLogger.

    The LogLevel of an log call can be defined in two was. The first is by calling logl and passing the LogLevel explicit. Notice the additional l after log. The LogLevel is to be passed as first argument to the function. The second way, of setting the LogLevel of a log call, is be call either trace, info, warning, critical or fatal. The log call will than have the respective LogLevel.

    Conditional logging can be achived be appending a c to the function identifier and passing a bool as first argument to the function. If conditional logging is used the condition must be true in order to have the log message logged.

    In order to combine a explicit LogLevel passing with conditional logging call the function or method loglc. The first required argument to the call then becomes the LogLevel and the second argument is the bool.

    Messages are logged if the LogLevel of the log message is greater equal than the LogLevel of the used Logger and additionally if the LogLevel of the log message is greater equal to the global LogLevel. The global LogLevel is accessible by using LogManager.globalLogLevel. To assign the LogLevel of a Logger use the logLevel property of the logger.

    If printf style logging is required add a f to the logging call, like

    such:
    myLogger.infof("Hello %s", "world"); or The additional f enables printf style logging for call combinations of explicit LogLevel and conditional logging functions and methods. The f is always to be placed last.

    To customize the logger behaviour, create a new class that inherits from the abstract Logger class, and implements the writeLogMsg method. 
    class MyCustomLogger : Logger {
    override void writeLogMsg(ref LoggerPayload payload))
    {
        // log message in my custom way
    }
}

auto logger = new MyCustomLogger();
logger.log("Awesome log message");

    enum LogLevel: ubyte;
    There are eight usable logging level. These level are all, trace, info, warning, error, critical, fatal, and off. If a log function with LogLevel.fatal is called the shutdown handler of that logger is called.

    unspecific
    If no LogLevel is passed to the log function this level indicates that the current level of the Logger is to be used for logging the message.

    all
    Lowest possible assignable LogLevel.

    trace
    LogLevel for tracing the execution of the program.

    info
    This level is used to display information about the program.

    warning
    warnings about the program should be displayed with this level.

    error
    Information about errors should be logged with this level.

    critical
    Messages that inform about critical errors should be logged with this level.

    fatal
    Log messages that describe fatel errors should use this level.

    off
    Highest possible LogLevel.

    abstract class Logger;
    This class is the base of every logger. In order to create a new kind of logger a deriving class needs to implement the writeLogMsg method.

    struct LoggerPayload;
    LoggerPayload is a aggregation combining all information associated with a log message. This aggregation will be passed to the method writeLogMsg.

    string file;
    the filename the log function was called from

    int line;
    the line number the log function was called from

    string funcName;
    the name of the function the log function was called from

    string prettyFuncName;
    the pretty formatted name of the function the log function was

    called from

    LogLevel logLevel;
    the LogLevel associated with the log message

    SysTime timestamp;
    the time the message was logged.

    string moduleName;
    the name of the module

    Tid threadId;
    thread id

    string msg;
    the message

    @safe this(string newName, LogLevel lv);
    This constructor takes a name of type string, and a LogLevel.

    Every subclass of Logger has to call this constructor from there constructor. It sets the LogLevel, the name of the Logger, and creates a fatal handler. The fatal handler will throw an Error if a log call is made with a LogLevel LogLevel.fatal.

    void writeLogMsg(ref LoggerPayload payload);
    A custom logger needs to implement this method.

    @trusted void logMessage(string file, int line, string funcName, string prettyFuncName, string moduleName, LogLevel logLevel, bool cond, string msg);
    This method is the entry point into each logger. It compares the given LogLevel with the LogLevel of the Logger, and the global . If the passed LogLevel is greater or equal to both the message, and all other parameter are passed to the abstract method writeLogMsg.

    class LogManager;
    The static LogManager handles the creation, and the release of instances of the Logger class. It also handles the defaultLogger which is used if no logger is manually selected. Additionally the LogManager also allows to retrieve Loggers by their names. An StdIOLogger is assigned to be the default Logger.

    static @property ref @trusted Logger defaultLogger();
    This method returns the default Logger.

    The Logger is returned as a reference. This means it can be rassigned, thus changing the defaultLogger.

    Example:
    LogManager.defaultLogger = new StdIOLogger;
    The example sets a new StdIOLogger as new defaultLogger.

    static @property @trusted LogLevel globalLogLevel();
    This method returns the global LogLevel.

    static @property @trusted void globalLogLevel(LogLevel ll);
    This method sets the global LogLevel.

    Every log message with a LogLevel lower as the global LogLevel will be discarded before it reaches writeLogMessage method.
